文档中心

从安装 OpenClaw、启动本地网关、接入微信机器人，到配置 Ollama 模型、知识库和自动化任务流。这里按真实落地顺序整理 FaroAI 的使用路径。

最后校对：2026-05-15。本机实测 OpenClaw 为 2026.5.7，npm registry latest 为 2026.5.12。命令示例按本站本机 OpenClaw Gateway 验证路径整理；不同版本参数可能变化，执行前以 openclaw --help 和官方文档为准。

快速开始

FaroAI 推荐从一个可运行的本地网关开始，再逐步接入模型、微信通道和自动化规则。这样每一步都能被验证，出问题时也容易定位。

确认 OpenClaw CLI 可用

先确认命令行能正常运行，并记录当前版本。后续所有配置、诊断和重启都通过 CLI 完成。

openclaw --version
openclaw doctor

启动 Gateway

Gateway 是 FaroAI 的本地中枢，负责连接模型、通道和任务调用。

openclaw gateway install
openclaw gateway restart
openclaw gateway status

验证模型调用

不要一上来接复杂通道，先确认 Gateway 能调用 AI 模型。

openclaw capability model run --gateway \
  --model ollama/qwen3.6:27b-mlx-bf16-fast \
  --prompt "请只回复 OK。" --json

安装与升级

如果你通过 npm 安装 OpenClaw，可以直接安装 latest。2026-05-15 查询到 npm latest 为 2026.5.12；升级后建议立即运行 doctor，并重启 Gateway。

全局安装

适合个人电脑、Mac Studio、本地开发机。安装后 CLI 会在终端中可用。

npm install -g openclaw@latest
openclaw --version

升级现有环境

已有安装时使用内置更新命令，它会根据当前安装方式执行更新。

openclaw update --yes
openclaw doctor --fix --yes --non-interactive
openclaw gateway restart

提示：不同版本的命令参数可能略有变化。遇到参数不识别时，先运行 openclaw <command> --help 查看当前版本支持的选项。

网关 Gateway

Gateway 是 FaroAI 的运行核心。你可以把它理解为一个本地 AI 总线：微信机器人、网页、CLI、模型 provider、工具调用都会经过这里协调。

常用健康检查

检查服务是否加载、端口是否监听、是否具备管理能力。

openclaw gateway status
openclaw channels status --probe
openclaw health

本地默认端口

常见本地配置为 loopback 绑定，端口 18789。仅本机访问更适合个人使用和本地模型。

ws://127.0.0.1:18789
http://127.0.0.1:18789/

微信机器人接入

微信通道让 FaroAI 进入你的日常聊天入口。接入后，你可以通过微信触发总结、搜索、长文整理、日程辅助和自动化任务。

安装微信插件

当前 FaroAI 使用 OpenClaw 插件机制加载微信通道。

openclaw plugins install @tencent-weixin/openclaw-weixin --force
openclaw plugins inspect openclaw-weixin --runtime --json

确认通道运行

状态中应看到 openclaw-weixin 为 enabled、configured、running。

openclaw gateway restart
openclaw channels status --probe

设置默认模型

微信通道可以走默认 agent 模型，也可以按消息规则路由到不同模型。

openclaw models list
openclaw agents list

模型配置

FaroAI 支持本地优先的模型策略。简单问题可以走快速模型，复杂分析、长文写作和代码任务可以路由到更强模型。

Ollama 本地模型

适合隐私敏感、成本可控、需要长期运行的个人 AI 工作台。

ollama list
openclaw models list

模型调用测试

每次改模型配置后，用一次最小测试确认链路可用。

openclaw capability model run --gateway \
  --prompt "用一句话说明当前模型可用。" --json

知识库与记忆

知识库适合放长期稳定资料，例如项目文档、个人简历、产品说明、客户材料和常用 SOP。记忆则适合保存偏好、上下文和长期协作线索。

适合放入知识库的内容

产品介绍、常见问答、流程文档、技术规范、项目背景、合同模板、邮件模板。

不建议放入知识库的内容

明文密码、长期有效 token、银行卡信息、身份证件、一次性验证码和未经授权的他人隐私数据。

自动化任务流

当一个任务反复出现，就可以把它从聊天指令沉淀成任务流。FaroAI 的目标是让 AI 不只是回答，还能持续执行、跟进和复盘。

新闻与资料摘要

微信发起主题，FaroAI 搜索资料、抽取重点、生成摘要，并按固定格式回复。

技术排障助手

收集日志、识别错误、给出下一步命令，并把操作过程整理成复盘。

内容创作流水线

输入主题后生成提纲、初稿、标题、摘要和发布检查清单。

个人知识助理

围绕你的知识库进行问答、补充背景、生成学习计划和行动项。

故障排查

优先按“网关是否在线 - 插件是否加载 - 通道是否运行 - 模型是否可调用”的顺序排查。不要直接改复杂配置，先用最小命令确认是哪一段链路失败。

Gateway 显示 not reachable 怎么办？

先运行 openclaw gateway status，如果服务未运行，执行 openclaw gateway restart。如果端口被占用，可查看 status 输出中的进程信息。

微信插件显示 unknown channel id 怎么办？

通常是插件未安装成功或有旧配置残留。重新安装插件后运行 openclaw plugins registry --refresh，再重启 Gateway。

模型调用慢或失败怎么办？

先用最小 prompt 测试模型。如果本地模型很大，确认 Ollama 已加载、机器资源足够，并适当增加 timeout。

页面登录为什么显示即将开放？

当前网站是静态展示站，Google/GitHub OAuth 后端还未接入。现在可先通过文档配置本地 FaroAI，或通过联系方式申请试用。
