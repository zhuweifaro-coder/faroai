API 参考文档

通过强大的 REST API 和 SDK，将 FaroAI 的 AI 能力集成到你的应用中。

🔑 认证方式

FaroAI API 使用 Bearer Token 进行认证。所有 API 请求需要在 Header 中携带有效的 API Key。

Authorization: Bearer your-api-key-here

你可以在控制台 → API 管理中创建和管理 API Key。建议为不同用途创建不同的 Key，并定期轮换以确保安全。

速率限制

所有 API 请求都有速率限制，具体如下：

方案请求/分钟请求/天并发连接
个人版105002
专业版6010,00010
企业版不限不限专属配额

当超过速率限制时，API 会返回 HTTP 429 状态码，并在响应头中包含 Retry-After 字段，指示下次可重试的时间。

📡 端点列表

智能体管理 (Agents)

POST /v1/agents

创建新的 AI 智能体。支持配置名称、模型、系统提示词、知识库关联等参数。

GET /v1/agents

列出所有智能体。支持分页和过滤，可按名称、状态、创建时间排序。

GET /v1/agents/:id

获取指定智能体的详细信息，包括配置、状态和最近活动记录。

PUT /v1/agents/:id

更新智能体配置，支持修改名称、模型、系统提示词、知识库等。

DELETE /v1/agents/:id

删除指定智能体。删除后不可恢复，同时会清除关联的工作流和数据。

消息收发 (Messages)

POST /v1/agents/:id/messages

向指定智能体发送消息。支持文本、图像等多模态输入，返回流式或非流式响应。

GET /v1/agents/:id/messages

获取智能体的消息历史记录，支持分页和时间范围过滤。

工作流管理 (Workflows)

POST /v1/workflows

创建新的自动化工作流，定义触发器、动作序列和条件逻辑。

POST /v1/workflows/:id/run

手动触发执行指定工作流，可选择传入自定义参数。

GET /v1/workflows/:id/status

查询工作流执行状态和进度，支持实时流式更新。

知识库管理 (Knowledge Bases)

POST /v1/knowledge-bases

创建新的知识库，支持设置名称、描述、关联智能体等。

POST /v1/knowledge-bases/:id/documents

向知识库添加文档。支持上传文件（PDF、Markdown、TXT、Word）或直接提交文本内容。

GET /v1/knowledge-bases/:id/search

在知识库中进行语义搜索，返回最相关的文档片段和相似度评分。

DELETE /v1/knowledge-bases/:id

删除知识库及其所有文档。操作不可逆，请谨慎使用。

📦 SDK

FaroAI 提供多语言 SDK，让你用最熟悉的语言快速集成 AI 能力。

🐍

Python SDK

完整的 Python 客户端，支持同步和异步调用

pip install faroai

from faroai import FaroAI
client = FaroAI("your-api-key")
response = client.chat.messages.create(
  model="gpt-4",
  messages=[{"role": "user", "content": "你好"}]
)
💚

Node.js SDK

JavaScript/TypeScript 客户端，支持 Promises 和 async/await

npm install faroai

import FaroAI from 'faroai';
const client = new FaroAI({
  apiKey: 'your-api-key'
});
const res = await client
  .chat.messages.create({
    model: 'gpt-4',
    messages: [{role: 'user', content: '你好'}]
  });
🔵

Go SDK

高性能 Go 客户端，原生支持并发和流式响应

go get github.com/faroai/go-sdk

🪝 Webhook 配置

Webhook 允许你在关键事件发生时收到实时通知。你可以在控制台配置 Webhook URL，FaroAI 会以 POST 请求方式推送事件数据。

支持的事件类型

workflow.completed - 工作流执行完成
workflow.failed - 工作流执行失败
message.received - 收到新消息
agent.updated - 智能体配置变更
kb.document.indexed - 知识库文档索引完成
rate_limit.warning - 接近速率限制阈值

Webhook 请求格式

POST https://your-server.com/webhook

{
  "event": "workflow.completed",
  "timestamp": "2026-05-05T06:00:00Z",
  "data": {
    "workflow_id": "wf_123456",
    "status": "success",
    "duration_ms": 3200
  }
}

所有 Webhook 请求都携带签名头 X-FaroAI-Signature，用于验证请求来源。建议在服务端验证签名以确保安全性。Webhook 会进行最多 5 次重试，重试间隔分别为 1 分钟、5 分钟、15 分钟、30 分钟和 1 小时。
