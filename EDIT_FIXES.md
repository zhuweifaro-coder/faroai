# 🔧 编辑工具使用最佳实践

## ❌ 问题记录

**错误原因**：`edit` 工具要求 `oldText` 与原文**完全精确匹配**（包括所有空格、换行、特殊字符）。

**出现场景**：
- 文件中有不可见字符
- 缩进字符混用（空格 vs tab）
- 换行符差异
- 特殊空白字符（如不间断空格）

## ✅ 解决方案

### 优先使用 `write` 重写整个文件

当文件较小（<5KB）或需要大量修改时：
```javascript
// ✅ 推荐：直接 write 重写
write(path, newContent)
```

### 使用 `edit` 时的注意事项

1. **先 read 确认原文**
2. **精确复制要替换的文本**
3. **注意空白字符和缩进**
4. **小范围编辑**（避免跨多行的大块文本）

### 调试步骤

```
1. read 文件查看内容
2. 使用 read + offset 查看特定行
3. 确认要替换的文本精确匹配
4. 如果不确定，直接用 write 重写
```

## 📝 本项目应用

本次修复：
- 原文件有不可见的特殊空格字符
- edit 无法精确匹配
- 改用 write 重写整个文件 → ✅ 成功

## 🚀 后续建议

对于 `worker.js` 这类小型 Worker 脚本：
- 文件 < 5KB
- 改动频繁
- **优先使用 write 重写**
- 避免使用 edit
